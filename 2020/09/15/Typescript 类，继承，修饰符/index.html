1 类的定义

class People{
    name:string
    // constructor 是类实例化触发的构造函数
    constructor(name:string){
        this.name = name
    }

    run(){
        console.log(this.name)
    }
    // 可以通过类中的方法去对属性改值，取值
    getName():string{
        return this.name
    }

    setName(name:string):void{
        this.name = name
    }
}

let peo = new People("hz")
peo.run()

2 继承

class People{
    name:string
    constructor(name:string){
        this.name = name
    }
    run(){
        console.log(this.name)
    }
}
// 继承：通过extend super
class Web extends People{
    age:number
    constructor(name:string,age:number){
        // 继承要点1：必须继承父类的构造方法，可以在上面新增，但是不能缺
        super(name)
        this.age = age
    }
    // 继承要点2：属性和方法的查找顺序优先从自己开始到父类
    run(){
        console.log(this.age)
    }
}

var w =new Web("hz",18)
w.run()

3 修饰符

修饰符修饰的是类中的属性和方法

  • public：公有
    • 在类中，子类中，类外部都可以访问
  • protected：保护类型（封装）
    • 在类中，子类中可以访问，类外部不能访问
  • private：私有
    • 只有在类中能访问，在子类和类外部都不能访问

注意：属性如果不加修饰符默认是public

class People{
    public name:string
    protected sex:boolean = false
    private xxx:number = 123
    constructor(name:string){
        this.name = name
    }
    run(){
        console.log(this.name,this.sex,this.xxx)
    }
    protected work(){
        console.log("x",this.xxx)
    }
}


class Web extends People{
    age:number
    constructor(name:string,age:number){
        // 继承要点1：必须继承父类的构造方法，可以在上面新增，但是不能缺
        super(name)
        this.age = age
    }
    // 继承要点2：属性和方法的查找顺序优先从自己开始到父类
    // 继承要点3：如果父类中有私有属性，子类中不能重复定义这个属性
    run(){
        console.log(this.age,this.sex)
    }
    work(){
        this.work()
    }
}

// 类中的修饰符运用
var w = new Web("hz",18)
w.run()
w.work()

4 静态属性/方法

class People{
    public name:string
    protected sex:boolean = false
    private xxx:number = 123
    static wifi:string = "lxt"
    constructor(name:string){
        this.name = name
    }
    // 实例方法,只能使用实例属性
    run(){
        console.log(this.name,this.sex,this.xxx)
    }
    protected work(){
        console.log("x",this.xxx)
    }
    // 静态方法，静态方法不能直接调用属性，只能调用静态属性
    static play(){
        console.log(this.wifi+"在play")
    }
}
var p = new People("hz")
People.play()
p.run()

5 多态

ts中的多态属于继承的一种形式，父类定义方法但不去实现，子类去实现父类的方法，实现多态

ts中的多态属于鸭子类型，和python和go中的多态相似，都属于非侵入式，就算不写也不会报错

class Animal{
    name:string
    constructor(name:string){
        this.name = name
    }
    eat(){
    }
}
class Dog extends Animal{
    constructor(name:string){
        super(name)
    }
    eat(){
        console.log(this.name+"吃肉")
    }
}

class Cat extends Animal{
    constructor(name:string){
        super(name)
    }
    eat(){
        console.log(this.name+"吃🐭")
    }
}
var dog = new Dog("xd")
dog.eat()

var cat = new Cat("xm")
cat.eat()

6 抽象类，抽象方法

注意点：

  • 抽象方法无法单独存在必须定义在抽象类中
  • 继承抽象类的子类，必须实现抽象类中的抽象方法
  • 抽象类无法实例化
  • 抽象类的继承也属于多态，属于侵入式，没有重写就会报错
abstract class Animal{
    name:string
    constructor(name:string){
        this.name = name
    }
    // 定义在抽象类内部的抽象方法
    // 不能在这里实现
    // 必须定义返回值类型：void为无返回值
    abstract eat():void
}
class Dog extends Animal{
    constructor(name:string){
        super(name)
    }
    eat(){
        console.log(this.name+"吃肉")
    }
}

class Cat extends Animal{
    constructor(name:string){
        super(name)
    }
    eat(){
        console.log(this.name+"吃🐭")
    }
}
var dog = new Dog("xd")
dog.eat()

var cat = new Cat("xm")
cat.eat()
